---
title: "Discovery framework"
---

<p className="lede">Five themes: agents, authority, incidents, evidence, fit. Get them to describe one specific agent and one action that would scare them. The rest follows.</p>

## Opening (map the surface)

<ul>
<li>Which agents or copilots can take actions in or near production today? Through which paths, MCP, an SDK, a gateway, direct APIs?</li>
<li>Who decides what each agent is allowed to <i>do</i> right now, on a specific resource? How is that enforced before the action?</li>
</ul>

## Authority & scope

<ul>
<li>How do you stop an in-scope, authenticated agent from doing something you'd never approve?</li>
<li>Do you have change freezes, PII/regulated scopes, or financial limits agents must respect? How enforced today, or is it convention?</li>
</ul>

## Incidents & evidence

<ul>
<li>Have you had an agent or automation make something worse, or surface in an audit? What happened?</li>
<li>How fast can you answer "who authorized this action, under which policy, and why"? Is that answer signed?</li>
</ul>

## Fit & motion

<ul>
<li>What would you be comfortable letting an agent auto-approve in non-prod vs prod?</li>
<li>Is EU data residency / sovereignty a requirement for this workload?</li>
</ul>

<Info>
<span>Tell.</span> When they can't name who decides the action (only who has access), you've found the action gap. Name it back to them.
</Info>
